【Configuration】repo:command not found for Mac -已解决

Posted on 2020-12-06，2 min read

欢迎来到我的小站呀，很高兴遇见你！(╹▽╹)

为什么会遇到这个问题？

最近在使用Mac尝试编译Android源码时遇到的错误

解决了吗？

当然，我的解决方案是在终端窗口输入下面两条指令

echo 'export PATH=$PATH:$HOME/bin' >> ~/.bash_profile
export PATH=$PATH:$HOME/bin

你是如何解决的？

网上普遍的做法是

echo 'export PATH=$PATH:$HOME/bin' >> ~/.bashrc  
export PATH=$PATH:$HOME/bin

这两行代码在我的电脑环境中只会临时有效，重新打开新终端窗口依旧报not found
环境变量有效范围不是全局的也不是用户的，这就很神奇了，我打算理解这两行指令

我们来看第一行指令

 echo 'export PATH=$PATH:$HOME/bin' >> ~/.bashrc

>>是追加内容操作，我们去用户目录下找到.bashrc这个文件
左上角访达->前往->前往文件夹->键入~/

进入目录后按下Command+Shift+. 显示隐藏文件

打开.bashrc文件

如上图所示，.bashrc文件的确已经追加PATH路径，那为什么不生效呢？
一番搜索后我发现因为OS X在login启动时不会加载.bashrc文件，它的读取顺序是

1. /etc/profile
2. ~/.bash_profile
3. ~/.bash_login
4. ~/.profile

既然已经知道问题所在了，那么解决起来就比较简单了

方案一： 在~/.bash_profile文件中追加
    if [ -f ~/.bashrc ]; then
        source ~/.bashrc
    fi
方案二：直接将PATH路径追加到.bash_profile中
终端输入：echo 'export PATH=$PATH:$HOME/bin' >> ~/.bash_profile

搞定，收工

有需要补充的吗？

文中第二条指令没有讲，它的作用就是设置当前终端的环境变量，只对当前窗口有效.
